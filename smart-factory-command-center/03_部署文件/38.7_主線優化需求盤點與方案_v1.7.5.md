# 38.7_主線優化需求盤點與方案｜v1.7.5

狀態：需求已收斂，39 暫停。

## 一、目前主線

```text
38.6 已完成
39 暫停
先做主線優化
PWA 為主入口
每日依據生產計劃表自動排程，派給作業員報工
```

## 二、使用者本次提出的核心問題

1. 主畫面圖示與首頁文案要改，點進去第一眼要是：NEXUS OS · 工業 5.0 智慧製造矩陣。
2. PWA 要成為主要入口。
3. 生產計劃表目前可清洗，但品名錯誤，會出現產品編號，需要自動修正。
4. 每天要知道人員要做什麼產品加工。
5. 報工目前有 GAS 與 PWA，要統一使用流程。
6. 報工作業機台照片無法顯示，需要修復。
7. LINE Bot 免費訊息有限，公告不應依賴大量推播，需用 HTML 公告頁或 PWA 公告中心。
8. 所有資料需要唯一資料庫與唯一鍵，不要每開專案又產生新試算表。
9. Notion 可作為讀取展示與管理看板，但 Google Sheets 仍是主資料庫。
10. LINE Bot 應提供 HTML 註冊頁，讓人員用工號綁定 LINE_USER_ID，註冊後可登入 PWA，也可讓 Bot 自動套用角色選單。
11. 排班規則必須修正：幹部不排班，學生不加班，專責人員要依專責規則派工。

## 四、資料唯一原則

主資料庫只能有一份：

```text
智慧製造中央作戰指揮中心資料庫
```

建議所有 GAS 與 PWA 只能讀同一個：

```text
PropertiesService: 智慧製造_SPREADSHEET_ID
或 docs/pwa-config.js: SPREADSHEET_ID
```

不可再由初始化流程自動建立新專案用新試算表。初始化只能修復既有分頁與欄位。

## 五、生產計劃表清洗修正方向

問題：品名欄出現產品編號。

修正規則：

```text
先讀生產計劃表原始列
取產品編號 / 客戶品號 / 品名候選
用 02_產品主檔 回補正確品名
再用 08_工站途程機台主檔 回補工站、機台、標準產能、標準工時
最後寫入 10_排程需求池
```

精準對應優先順序：

```text
產品編號 + 客戶品號
產品編號
客戶品號
品名模糊比對只做提示，不自動覆蓋
```

## 六、每日自動排程目標

每天產出：

```text
今日要加工產品
對應工站
對應機台
需求數量
建議作業員
班別
是否可加班
報工入口
```

輸出到：

```text
20_今日派班
21_今日派班報工對接
PWA 首頁今日任務
LINE Bot 查詢：我的狀態 / 今日任務
```

## 七、排班規則修正

硬性規則：

```text
幹部不排班
工程師 / 主任 / 助理工程師預設不進一般作業員排程
學生不加班
留職停薪不排班
不加班人員不可排加班
專責人員優先派專責工站或產品
```

本次明確人員規則需要建成資料表，不可只寫在程式裡。

建議新增分頁：

```text
19_人員排班規則
```

欄位：

```text
工號
姓名
角色類型
預設班別
可排班
可加班
專責工站
專責產品
專責機台
限制工站
限制產品
限制機台
備註
更新時間
```

## 八、報工統一策略

PWA 為主要入口，GAS HTML 保留為備援與內嵌來源。

正式流程：

```text
PWA 首頁
→ 今日任務
→ 點選任務
→ 帶入工號、產品、工站、機台、工單
→ 送出報工
→ 寫入 09_報工
→ 回扣 10_工單主檔 / 20_今日派班 / 21_今日派班報工對接
```

## 九、機台照片修復方向

## 十、LINE Bot 訊息成本策略

LINE 免費訊息有限，不用大量推播公告。

做法：

```text
重大通知：LINE 推一則短訊息 + PWA/HTML公告連結
一般公告：PWA 公告中心
每日任務：作業員主動查詢或點 Rich Menu
主管戰情：主管主動查詢 + 必要提醒
```

## 十一、LINE 註冊 HTML

建議新增：

```text
LINE人員註冊.html
```

流程：

```text

```

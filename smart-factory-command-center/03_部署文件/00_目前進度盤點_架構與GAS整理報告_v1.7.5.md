## 二、目前整體架構

```text
LINE 使用者
  ↓
Rich Menu 主管入口 / 報工入口
  ↓
LINE Webhook doPost(e)
  ↓
37_LINE 指令中心
  ↓
34_LINE 角色選單分流
  ↓
33_LINE 身份權限檢查
  ↓
31_LINE 日期快選
  ↓
30_LINE 主管戰情直連
  ↓
一般 LINE Webhook / API Handler
  ↓
Google Sheets 中央作戰資料庫
```

目前系統分成五大層：

| 層級 | 說明 | 代表檔案 |
|---|---|---|
| 主入口層 | doGet / doPost / API 分流 | 智慧製造中央作戰指揮中心.gs |
| 報工層 | PWA 報工、派班、日結、防重複 | 07、09、21～24 |
| 戰情層 | 主管戰情 API、看板、LINE 回覆 | 25～31 |
| LINE 權限與選單層 | Rich Menu、身份綁定、角色分流、指令中心 | 32～38 |
| 維護層 | 自檢、觸發器、異常修復、選單修復 | 系統維護工具、27、36 |

## 三、目前關鍵事實與風險

### 1. GitHub 主檔 doPost 可能尚未合併到最新 37/38 接線

`智慧製造中央作戰指揮中心.gs` 目前 GitHub 版標頭仍顯示 v1.6.8，主 doPost 目前只看得到 31 日期快選、30 主管戰情直連與一般 webhook，尚未在主檔內看到 37、34、33 的最新接線順序。

目前正確接線應為：

```text
37_LINE 指令中心
↓
34_LINE 角色選單分流
↓
33_LINE 身份權限檢查
↓
31_LINE 主管戰情日期快選
↓
30_LINE 主管戰情直連
↓
一般 LINE Webhook
```

因此後續整理第一優先不是刪檔，而是確認：

```text
Apps Script 實際執行版 doPost 是否已經是 37 版
GitHub 的 智慧製造中央作戰指揮中心.gs 是否同步更新
```

### 2. 26 與 39 不能重做

`26_LINE每日戰情推播正式模組.gs` 已存在。  
後續若進入 39，應做：

```text
39_LINE 每日戰情推播升級與主管提醒補強
```

不是重做每日推播。

26 是主引擎；39 只能做檢查、補強、紀錄、重送、提醒規則。

### 3. 32～38 不可直接刪

32～38 是目前 LINE 權限、選單、指令中心的正式主線。即使部分看起來是工具檔，也仍被目前流程使用。

## 七、目前不要刪的原因

現在不要直接刪檔，原因：

1. GAS 目前是單一命名空間，刪掉任何函數都可能造成其他模組 `typeof xxx === 'function'` 檢查失效。
2. 26、27 是未來 39 推播補強的基礎。
3. 34、36 依賴 33 身份權限表。
4. 38 依賴 34 的 `批次同步34_LINE所有已綁定使用者選單()`。
5. 37 doPost 是目前最新 webhook 分流基準。

# 37_LINE｜指令中心與快捷指令總表｜部署 SOP

## 一、模組定位

第 37 階段只做 LINE 指令中心與快捷指令總表。

本階段不修改：

- 33_LINE 身份權限判斷
- 34_LINE 使用者角色分流
- 35_LINE 一般員工 Rich Menu 圖片
- 36_LINE 選單維護工具
- 報工 PWA
- 自動排程

## 二、已新增 GAS 檔案

```text
smart-factory-command-center/01_GAS後端/37_LINE_指令中心與快捷指令總表.gs
smart-factory-command-center/01_GAS後端/37_LINE_主後端doPost正式替換段.gs
```

## 三、會新增工作表

| 工作表 | 用途 |
|---|---|
| 37_LINE指令中心 | 指令分類、指令文字、權限、說明、範例 |
| 37_LINE指令使用紀錄 | 記錄使用者查詢指令中心的紀錄 |

## 四、主後端接線

將 `智慧製造中央作戰指揮中心.gs` 內原本的 `doPost(e)` 整段替換為：

```text
smart-factory-command-center/01_GAS後端/37_LINE_主後端doPost正式替換段.gs
```

接線順序：

```text
37_LINE 指令中心
↓
34_LINE 角色選單分流
↓
33_LINE 身份權限檢查
↓
31_LINE 戰情日期快選
↓
主管戰情直連
↓
一般 LINE Webhook
```

## 五、部署順序

1. 將 `37_LINE_指令中心與快捷指令總表.gs` 加入 Apps Script。
2. 用 `37_LINE_主後端doPost正式替換段.gs` 替換主後端 `doPost(e)`。
3. 執行：

```javascript
初始化37_LINE指令中心與快捷指令總表()
```

4. 執行：

```javascript
測試37_LINE指令中心_本機規格()
```

## 六、LINE 測試指令

| 指令 | 預期回覆 |
|---|---|
| 指令 | 智慧製造指令中心總覽 |
| 主管指令 | 主管戰情、今日戰情、昨日戰情、主檔檢查、AI摘要 |
| 員工指令 | 報工作業、權限檢查、選單更新、綁定工號 |
| 系統維護 | 主管選單、員工選單、選單說明、我的狀態 |
| 我的狀態 | 姓名、工號、角色、權限、目標選單 |
| 選單說明 | 依身份回覆主管入口或報工入口說明 |

## 七、驗收標準

| 項目 | 標準 |
|---|---|
| 工作表 | 出現 `37_LINE指令中心`、`37_LINE指令使用紀錄` |
| 指令總表 | 至少 10 筆以上預設指令 |
| LINE 輸入 `指令` | 回覆總覽 |
| LINE 輸入 `主管指令` | 回覆主管指令清單 |
| LINE 輸入 `員工指令` | 回覆員工指令清單 |
| LINE 輸入 `我的狀態` | 回覆目前身份、角色與目標選單 |
| 原功能 | 主管戰情、報工作業、選單更新仍正常 |

## 八、注意事項

`37_LINE指令中心` 是可維護表。若未來新增指令，只要在表內新增一列，分類填 `主管`、`員工` 或 `維護`，並設定 `啟用=是`。

37 只負責查詢與說明，不負責執行主管戰情或報工作業；實際執行仍由原本 31、33、34、報工模組處理。

# 35_LINE｜一般員工 Rich Menu 圖片正式化｜部署 SOP

## 一、模組定位

第 35 階段只處理一般員工 Rich Menu 圖片正式化。

本階段不修改：

- 33_LINE 身份權限判斷
- 34_LINE 使用者角色分流
- 報工 PWA
- 自動排程
- 報工扣工單

## 二、正式圖片規格

| 項目 | 規格 |
|---|---|
| 圖片名稱 | 35_LINE_RichMenu_一般員工入口_正式圖.png |
| 圖片尺寸 | 1200 x 810 |
| 圖片格式 | PNG |
| 圖片用途 | LINE_RICH_MENU_一般員工圖片網址 |
| 下方選單文字 | 報工入口 |

## 三、正式圖片內容

六宮格如下：

| 位置 | 主標題 | 副標題 | 按鈕文字 |
|---|---|---|---|
| 左上 | 報工作業 | 掃碼 / 填報 | 開啟報工 |
| 中上 | 我的身份 | 權限確認 | 查看身份 |
| 右上 | 選單更新 | 重整入口 | 更新選單 |
| 左下 | 報工入口 | 現場入口 | 開始報工 |
| 中下 | 指令說明 | 可用指令 | 查看指令 |
| 右下 | 身份綁定 | 綁定工號 | 綁定身份 |

## 四、上線方式

1. 將 `35_LINE_RichMenu_一般員工入口_正式圖.png` 上傳到 Google Drive。
2. 分享權限改成：知道連結的任何人可查看。
3. 取得 Google Drive 檔案 ID。
4. 組成圖片直連網址：

```text
https://drive.google.com/uc?export=download&id=你的檔案ID
```

5. 到 Apps Script → 專案設定 → 指令碼屬性，設定：

```text
LINE_RICH_MENU_一般員工圖片網址 = 上面的圖片直連網址
```

## 六、LINE 測試

### 工程師 / 主管

輸入：

```text
選單更新
```

預期：

```text
目標選單：主管入口
```

### 一般員工

輸入：

```text
選單更新
```

預期：

```text
目標選單：一般員工入口
```

### 強制切一般員工入口

輸入：

```text
員工選單
```


## 八、注意事項

若執行 `建立34_LINE一般員工RichMenu()` 多次，會建立新的 Rich Menu ID，這是正常現象。新的 ID 會覆蓋 `LINE_RICH_MENU_一般員工_ID`。

若 LINE 畫面沒有立刻更新，請重新進聊天室或等待 30 秒到 1 分鐘。

# v3.0.0 正式部署交付文件

## 一、系統定位

系統名稱：

```text
智慧製造中央作戰指揮中心 v3.0.0
```

系統架構：

```text
GitHub Pages HTML5 前端
+ Google Apps Script API 後端
+ Google Sheets 智慧工廠主資料庫
+ LINE Bot 單一入口
```

---

## 二、正式部署順序

### 1. Apps Script 檔案貼入順序

請在同一個 Apps Script 專案中貼入所有 `01_GAS後端` 檔案。

最後入口必須使用：

```text
v3_0_GAS主控入口與LINE最終整合版.gs
```

若舊專案已有 `doGet(e)` 或 `doPost(e)`，請以 v3.0 主控入口為準，避免重複定義。

---

### 2. 初始化正式資料表

在 Apps Script 函數選單依序執行：

```text
初始化_v3_0_正式上線()
```

若要分段執行，也可依序執行：

```text
初始化系統()
升級_v2_1_欄位結構()
升級_v2_2_CTB齊套缺料欄位()
升級_v2_4_共用素材主檔與紀錄表()
升級_v2_6_跨部門待辦任務表()
升級_v2_9_任務自動推播設定表()
```

---

### 3. 部署 GAS Web App

部署設定：

```text
執行身分：我
存取權限：任何知道連結的人
```

取得 Web App URL 後，貼到 GitHub Pages 各頁面的 GAS Web App URL 欄位。

---

### 4. LINE Webhook 設定

LINE Webhook URL 使用 GAS Web App URL。

LINE 回覆入口統一使用：

```text
產生LINE回覆訊息_v3_0_(文字)
```

v3.0 支援指令：

```text
版本
系統版本
健康檢查
系統狀態
總檢查
上線檢查
```

並會自動向下相容 v2.8 / v2.7 / v2.5 / v2.4 / 舊主選單指令。

## 四、正式上線測試

### 1. 健康檢查

開啟：

```text
docs/system-test.html
```

貼上 GAS Web App URL，執行全部測試。

### 2. GAS API 測試

測試 action：

```text
健康檢查
系統總檢查_v3_0
取得KPI戰情
取得AI摘要
取得_v2_7_任務統計儀表板資料
取得_v2_8_任務負責人績效統計
取得_v2_9_任務週月完成率統計
```

### 3. LINE 測試

在 LINE Bot 輸入：

```text
版本
健康檢查
上線檢查
任務摘要
任務週報
低庫存
CTB圖卡
```

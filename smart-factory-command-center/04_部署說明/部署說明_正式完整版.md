# 部署說明｜智慧製造中央作戰指揮中心 正式完整版

## 一、部署目標

本專案採用：

```text
Google Sheets 主資料庫
+ Google Apps Script 後端 API
+ GitHub Pages HTML5 PWA 前端
+ LINE Bot 單一入口
```

不使用 PostgreSQL、Redis、Node 後端，降低現場維護難度。

---

## 二、Google Sheets 主資料庫

1. 開啟 `⭐智慧工廠主資料庫`。
2. 進入：擴充功能 → Apps Script。
3. 新增或貼上 `01_GAS後端/智慧製造中央作戰指揮中心.gs`。
4. 儲存。
5. 執行函數：

```text
初始化_智慧製造中央作戰指揮中心
```

第一次執行需要授權。

成功後會建立以下資料表：

- 00_系統設定
- 01_人員主檔
- 02_產品主檔
- 03_機台主檔
- 04_工站主檔
- 05_共用資料
- 09_報工紀錄
- 09_停機紀錄
- 09_換刀紀錄
- 09_不良紀錄
- 10_工單主檔
- 10_排程需求池
- 11_檢具主檔
- 12_AI分析紀錄
- LINE_指令設定
- 系統_操作紀錄

## 三、GAS Web App 部署

進入 Apps Script：

1. 右上角「部署」。
2. 選擇「新增部署作業」。
3. 類型選「網頁應用程式」。
4. 執行身分：我。
5. 存取權限：知道連結的任何人。
6. 部署。
7. 複製 Web App URL。

Web App URL 格式：

```text
https://script.google.com/macros/s/部署ID/exec
```

## 四、GitHub Pages 前端

GitHub repo 內已建立：

```text
docs/index.html
docs/app.html
docs/manifest.webmanifest
docs/service-worker.js
```

啟用 GitHub Pages：

1. GitHub repo → Settings。
2. Pages。
3. Source 選 `Deploy from a branch`。
4. Branch 選 `main`。
5. Folder 選 `/docs`。
6. Save。

啟用後開啟：

```text
https://qhero70.github.io/smart-factory-worker-app/app.html
```

打開後到「設定」頁，貼上 GAS Web App URL，按「儲存設定」。

## 六、標準測試流程

1. 執行 GAS 初始化。
2. 建立一筆測試工單。
3. 打開 GitHub Pages app。
4. 貼上 GAS Web App URL。
5. 測試連線。
6. 新增報工。
7. 回 Google Sheets 檢查 `09_報工紀錄`。
8. 檢查 `10_工單主檔` 已完成量是否增加。
9. LINE 輸入「戰情」測試回覆。

## 七、常見問題

### 1. 前端顯示尚未連線

代表尚未貼 GAS Web App URL，或 GAS 部署權限不是「任何知道連結的人」。

### 2. CORS 錯誤

GitHub Pages 呼叫 GAS 時，前端使用 `Content-Type: text/plain` 避免預檢請求。若仍錯誤，重新部署 GAS Web App。

### 3. Google Sheets 沒有資料表

代表尚未執行：

```text
初始化_智慧製造中央作戰指揮中心
```

### 4. LINE 不回覆

檢查：

- Webhook URL 是否為 GAS Web App URL。
- LINE_CHANNEL_ACCESS_TOKEN 是否已填入。
- GAS Web App 是否允許外部存取。

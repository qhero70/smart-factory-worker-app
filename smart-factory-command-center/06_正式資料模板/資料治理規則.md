# 正式資料治理規則

## 一、主鍵規則

每張主檔必須有唯一主鍵，不可空白、不可重複。

| 工作表 | 主鍵欄位 |
|---|---|
| 01_人員主檔 | 工號 |
| 02_產品主檔 | 產品編號 |
| 03_機台主檔 | 機台編號 |
| 04_工站主檔 | 工站代碼 |
| 10_工單主檔 | 工單編號 |
| 11_檢具主檔 | 檢具編號 |

---

## 二、啟用規則

主檔資料不建議直接刪除。

若資料暫時不用：

```text
啟用 = 否
```

若欄位有 `狀態`：

```text
狀態 = 停用
```

---

## 三、產品分類規則

依產品編號末碼判斷：

| 末碼 | 分類 |
|---|---|
| 0 | 成品 |
| 1 | 素材 |
| 8 | 半成品 / 待組 / 待精加工 |

---

## 四、機台狀態規則

建議固定使用：

- 可生產
- 保養
- 維修
- 停用
- 報廢

報廢機台不可出現在報工選單。

---

## 五、工單狀態規則

建議固定使用：

- 待生產
- 進行中
- 已完成
- 暫停
- 取消

新增報工後，系統會依良品數自動更新工單已完成量。

---

## 六、CSV 匯入規則

1. 第一列必須是欄位名稱。
2. 欄位名稱必須與工作表欄位完全一致。
3. CSV 允許少欄，但不可出現不屬於該工作表的欄位。
4. 追加匯入不會清空原資料。
5. 覆蓋匯入會清空第 2 列以下所有資料。
6. 正式上線前建議先建立備份。

---

## 七、資料維護流程

正式建議流程：

```text
建立備份
→ 匯入或新增資料
→ 一鍵自檢
→ 修復欄位
→ 戰情室同步確認
```

---

## 八、不可跳步原則

正式資料尚未完成前，不建議進入：

- 生產計畫表清洗
- 自動產生工單
- 自動排程
- LINE Bot 主選單正式開放

原因：主檔不穩，後續排程與報工分析都會錯。

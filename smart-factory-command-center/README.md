# 智慧製造中央作戰指揮中心

> 正式版主線專案。此目錄不沿用舊的 PostgreSQL / Redis / Expo 架構，而是改以嘉欣現場最容易落地的架構建置：**Google Sheets + Google Apps Script + HTML5 PWA + LINE Bot 單一入口**。

---

## 一、系統定位

本系統定位為製造部的「中央作戰指揮中心」，不是單一報工表單。

主要目標：

1. 現場人員可用手機直接報工。
2. 管理者可即時看到產出、停機、不良、換刀、工單狀態。
3. 所有資料先集中寫入 Google Sheets，作為智慧工廠主資料庫。
4. LINE Bot 只保留一個入口，所有通知、查詢、工單、異常都從同一入口進入。
5. 後續可再延伸到自動排程、檢具管理、AI 交期風險分析。

## 四、使用方式摘要

1. 建立或打開 `⭐智慧工廠主資料庫` Google 試算表。
2. 開啟 Apps Script。
3. 貼上 `01_GAS後端/智慧製造中央作戰指揮中心.gs`。
4. 若要把 HTML 放在 GAS 同一專案，新增 `index.html`，貼上 `02_HTML前端/index.html`。
5. 執行 `初始化_智慧製造中央作戰指揮中心()`。
6. 部署為 Web App。
7. 將 Web App URL 貼到 GitHub Pages 前端的「設定」頁。
8. 測試：健康檢查、取得首頁資料、新增報工、取得工單清單。

## 五、正式驗收標準

正式版不是只看畫面，要用以下標準驗收：

- 能自動建立所有工作表。
- 能寫入報工紀錄。
- 能寫入停機紀錄。
- 能寫入換刀紀錄。
- 能寫入不良紀錄。
- 新增報工後，工單完成量會同步更新。
- 首頁能看到今日良品、今日不良、今日報工筆數。
- GitHub Pages 前端能連到 GAS Web App。
- LINE Bot Webhook 能回應基本查詢。

## 六、重要原則

- 不用 PostgreSQL。
- 不用 Redis。
- 不用複雜雲端後端。
- 所有欄位名稱、分頁名稱、函數名稱、註解優先使用繁體中文。
- 先讓現場能用，再逐步補強自動排程與 AI 分析。

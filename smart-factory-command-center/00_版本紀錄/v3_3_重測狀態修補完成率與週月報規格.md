# v3.3.0 重測狀態、修補完成率與週/月報規格

## 一、版本定位

```text
v3.3.0 = 上線修補追蹤與驗收版
```

此版將 v3.2 的修補建議、GitHub Issue 與 Commit 對照，進一步補上：

```text
正式建立 GitHub Issue 前端按鈕
重測清單狀態更新頁
修補完成率儀表板
上線修補週報 / 月報
```

---

## 二、新增 GAS 檔案

```text
01_GAS後端/v3_3_重測狀態修補儀表板與週月報工具.gs
```

新增函數：

```text
正式建立_v3_3_GitHubIssue(修補編號)
更新_v3_3_重測狀態(重測編號, 重測狀態, 重測結果, 備註)
取得_v3_3_修補完成率儀表板資料()
產生_v3_3_上線修補週報()
產生_v3_3_上線修補月報()
```

---

## 三、新增前端頁面

### 1. 修補重測狀態頁

```text
docs/retest-board.html
```

功能：

- 讀取 `00_修補重測清單`。
- 顯示重測總數、通過、失敗、待測、通過率。
- 可將重測項目更新為：通過、失敗、免測。
- 可輸入重測結果與備註。
- 可複製重測報告。

### 2. 修補完成率儀表板

```text
docs/fix-dashboard.html
```

功能：

- 錯誤總數。
- 修補任務總數。
- 修補完成率。
- 重測總數。
- 重測通過率。
- GitHub 對照數。
- 修補狀態圖。
- 重測狀態圖。
- 修補類型統計。
- 產生上線修補週報。
- 產生上線修補月報。

## 四、上線修補看板升級

更新：

```text
docs/fix-board.html
```

每張修補卡新增：

```text
正式Issue
```

若 `00_系統設定` 中有：

```text
GITHUB_TOKEN
GITHUB_REPO_FULL_NAME
```

系統會嘗試正式建立 GitHub Issue。

若未設定 `GITHUB_TOKEN`，仍會回傳 Issue 草稿，不中斷流程。

## 七、正式使用流程

```text
1. system-test.html 執行全部測試
2. 失敗項目回報
3. fix-board.html 產生修補建議
4. 建立 Issue 草稿或正式 Issue
5. 修補程式後登記 Commit
6. 建立重測清單
7. retest-board.html 更新重測結果
8. fix-dashboard.html 查看完成率與週/月報
9. 回 system-test.html 重新測試
```

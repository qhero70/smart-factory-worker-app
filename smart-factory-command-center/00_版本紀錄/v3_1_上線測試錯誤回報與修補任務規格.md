# v3.1.0 上線測試錯誤回報與修補任務規格

## 一、版本定位

```text
v3.1.0 = 正式部署後實測修補工具版
```

此版目的不是新增營運功能，而是讓正式上線後的錯誤可以被記錄、分類、建立修補任務、追蹤處理狀態。

---

## 二、本版新增內容

### 1. GAS 錯誤回報與修補任務工具

新增：

```text
01_GAS後端/v3_1_上線測試錯誤回報與修補任務工具.gs
```

功能：

- 建立上線測試錯誤回報表。
- 建立上線修補任務表。
- 接收 system-test 失敗項目。
- 自動判斷錯誤等級。
- 自動判斷修補類型。
- 自動建立修補任務。
- 更新修補任務狀態。
- 產生上線修補報告。

---

### 2. 新增資料表

#### 00_上線測試錯誤回報

欄位：

```text
錯誤編號、回報時間、來源頁面、測試類型、測試項目、錯誤等級、錯誤狀態、錯誤訊息、GAS網址、使用者、備註、原始資料、更新時間
```

#### 00_上線修補任務

欄位：

```text
修補編號、建立時間、來源錯誤編號、修補類型、負責人、優先級、修補狀態、問題摘要、建議處理、處理紀錄、完成時間、更新時間
```

---

### 3. 全頁測試頁升級

更新：

```text
docs/system-test.html
```

新增：

- v3.1.0 標示。
- 測試 `fix-board.html` 是否存在。
- 測試 `取得_v3_1_上線修補總覽` API。
- 新增「回報失敗項目」按鈕。
- 失敗項目可批次寫入 `00_上線測試錯誤回報`。
- 回報後自動建立 `00_上線修補任務`。

---

### 4. 新增上線修補看板

新增：

```text
docs/fix-board.html
```

功能：

- 查看錯誤回報。
- 查看修補任務。
- 依狀態、優先級、修補類型篩選。
- 更新修補任務狀態。
- 輸入處理紀錄與負責人。
- 產生並複製上線修補報告。

---

### 5. v3 主控入口升級

更新：

```text
01_GAS後端/v3_0_GAS主控入口與LINE最終整合版.gs
```

新增納入：

```text
升級_v3_1_上線測試錯誤表()
取得_v3_1_上線修補總覽()
回報_v3_1_上線測試錯誤()
```

LINE 新增指令：

```text
修補報告
上線修補
```

## 三、錯誤等級判斷

| 等級 | 條件 |
|---|---|
| 高 | doPost / doGet / Webhook / LINE / 權限 / Exception / TypeError / ReferenceError / GAS / API / 系統總檢查 |
| 中 | 404 / 缺少 / 欄位 / 工作表 / 資料 |
| 低 | 其他一般錯誤 |

---

## 四、修補類型判斷

| 類型 | 條件 |
|---|---|
| LINE Webhook | LINE / Webhook |
| GAS API | API / GAS / doPost / doGet / ReferenceError / TypeError |
| GitHub Pages | 頁面 / 404 / html |
| 資料表欄位 | 欄位 / 工作表 / 缺少 |
| 一般修補 | 其他 |

---

## 五、正式使用流程

```text
1. 開啟 docs/system-test.html
2. 貼上 GAS Web App URL
3. 按「執行全部測試」
4. 若有失敗，按「回報失敗項目」
5. 開啟 docs/fix-board.html
6. 依修補任務逐項處理
7. 完成後更新狀態
8. 再回 system-test 重新測試
```

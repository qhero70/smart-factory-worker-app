# v3.6.0 正式交付總包、合併 PDF、交接 PDF 與健康檢查排程規格

## 一、版本定位

```text
v3.6.0 = 正式交付總包與部署後維運版
```

此版將 v3.5 的正式交接結果進一步封裝成可保存、可下載、可交付、可部署後追蹤的正式交付總包。

完整流程：

```text
正式部署手冊
+ 正式上線封版報告
+ 正式上線交接清單
+ 版本鎖定紀錄
+ 健康檢查紀錄
→ 合併 PDF
→ 交接清單 PDF
→ 正式上線總包 ZIP
→ 部署後每日健康檢查排程
```

---

## 三、新增資料表

### 1. 00_正式交付總包紀錄

欄位：

```text
總包編號、建立時間、系統版本、總包類型、總包狀態、檔名、檔案連結、內容摘要、備註、更新時間
```

用途：

- 保存合併 PDF、交接清單 PDF、正式上線總包 ZIP 的產生紀錄。
- 保存 Google Drive 檔案連結。
- 作為正式交付檔案清單。

---

### 2. 00_部署後健康檢查排程

欄位：

```text
排程編號、建立時間、排程名稱、排程狀態、執行頻率、執行時間、觸發器ID、通知方式、最後執行時間、備註、更新時間
```

用途：

- 記錄每日健康檢查排程。
- 記錄觸發器 ID。
- 支援停用健康檢查排程。

---

### 3. 00_部署後健康檢查紀錄

欄位：

```text
紀錄編號、執行時間、系統版本、健康狀態、系統總檢查結果、缺少工作表數、缺少函數數、錯誤訊息、通知結果、備註、更新時間
```

用途：

- 保存每次部署後健康檢查結果。
- 記錄缺少工作表數、缺少函數數與錯誤訊息。
- 作為部署後維運追蹤依據。


## 八、健康檢查排程規則

每日健康檢查預設：

```text
每日 08:00 執行
```

執行函數：

```text
執行_v3_6_部署後每日健康檢查
```

檢查來源：

```text
系統總檢查_v3_0_()
```

健康狀態判斷：

```text
缺少工作表數 = 0
且
缺少函數數 = 0
且
系統總檢查成功 = true
→ 正常

否則 → 異常
```

---

## 九、正式使用流程

```text
1. 執行 初始化_v3_0_正式上線()
2. 完成 system-test / fix / retest / release / handoff
3. 開啟 docs/delivery-board.html
4. 匯出合併 PDF
5. 匯出交接清單 PDF
6. 產生正式上線總包 ZIP
7. 立即執行一次部署後健康檢查
8. 建立每日健康檢查排程
9. 用 LINE 輸入「交付狀態」確認統計
10. 用 LINE 輸入「每日健康檢查」立即檢查
```

---

## 十、注意事項

```text
PDF 匯出需要 Google Docs / Google Drive 權限
ZIP 產生需要 Google Drive 權限
每日健康檢查排程需要 Apps Script 觸發器權限
LINE 推播需既有 LINE 推播函數與目標 ID 設定
正式交付總包 ZIP 是交接資料包，不是 GitHub 原始碼完整備份
```

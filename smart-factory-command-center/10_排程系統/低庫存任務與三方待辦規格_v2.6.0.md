# 低庫存任務與三方待辦規格 v2.6.0

## 一、目的

v2.6.0 將低庫存從「看得到」升級為「自動建立任務、分派責任、追蹤處理」。

核心流程：

```text
共用素材主檔安全庫存
→ 低庫存掃描
→ 自動建立跨部門待辦任務
→ 生管 / 採購 / 製造三方看板追蹤
→ LINE 低庫存推播
```

---

## 二、新增資料表

### 10_跨部門待辦任務

欄位：

```text
任務編號、建立時間、任務來源、任務類型、負責單位、負責人、優先級、任務狀態、
共用素材編號、共用素材名稱、目前庫存、安全庫存、可分配數量、缺口數量、
最近需求交期、採購前置天數、最晚採購日、建議動作、處理紀錄、完成時間、更新時間
```

---

## 四、低庫存任務建立規則

當 `10_共用素材主檔` 中：

```text
目前庫存 <= 安全庫存
或 可分配數量 <= 0
```

且尚未有未完成的同素材「低庫存補料」任務時，系統自動建立任務。

---

## 五、優先級規則

系統會依 `10_排程需求池` 中同一個共用素材的最近需求交期判斷 Tier：

```text
Tier 1 → 優先級：高
Tier 2 → 優先級：中
Tier 3 / Tier 4 → 優先級：一般
```

---

## 六、最晚採購日

公式：

```text
最晚採購日 = 最近需求交期 - 採購前置天數
```

用途：

- 採購知道最晚何時必須下單或催料。
- 生管知道是否需要調整排程。
- 製造知道是否可以投產或需轉待料。

---

## 七、新增前端頁面

```text
docs/task-board.html
```

功能：

- 生管 / 採購 / 製造三欄任務看板。
- 一鍵掃描低庫存並建立任務。
- 任務狀態切換：待處理、處理中、完成、暫停。
- 可輸入處理紀錄與負責人。
- 產生任務摘要並複製。

---

## 八、LINE 推播

函數：

```text
推播_v2_6_LINE低庫存清單(目標ID)
```

注意：

- 需要 `LINE_CHANNEL_ACCESS_TOKEN`。
- 需要 LINE 目標 ID，例如 userId / groupId。
- 若未提供目標 ID，函數會回傳文字預覽，不會推播。

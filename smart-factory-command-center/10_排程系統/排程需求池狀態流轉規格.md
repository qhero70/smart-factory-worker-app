# 排程需求池狀態流轉規格

## 一、目的

將 `10_排程需求池` 從靜態資料表，變成可追蹤狀態的排程看板。

---

## 二、狀態定義

| 狀態 | 說明 | 負責確認 |
|---|---|---|
| 待排程 | 需求已產生，但尚未確認工站、機台、人力與交期 | 生管 / 製造 |
| 已排程 | 已決定工站、機台、人力或生產順序 | 生管 / 製造 |
| 生產中 | 已開始生產，應每日回報良品、不良、停機 | 製造 |
| 完成 | 生產需求已完成 | 製造 / 生管 |
| 暫停 | 因待料、設備、品質、交期調整等原因暫停 | 生管 / 製造 / 相關單位 |

---

## 三、建議流轉

```text
待排程 → 已排程 → 生產中 → 完成
```

異常時：

```text
待排程 / 已排程 / 生產中 → 暫停
暫停 → 待排程 或 已排程 或 生產中
```

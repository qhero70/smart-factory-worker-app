# 生產計畫清洗與排程需求池規格

## 一、目前 v1.4.0 目標

先建立第一版可落地流程：

```text
標準計畫 CSV
→ 清洗檢查
→ 產生工單
→ 產生排程需求池 CSV
```

此版先處理標準化後的 CSV，不直接解析複雜合併儲存格 Excel。

原因：正式主檔尚需穩定，先用標準 CSV 建立可驗證流程，再回頭支援複雜原始計畫表。

---

## 二、標準計畫 CSV 欄位

| 欄位 | 必填 | 說明 |
|---|---|---|
| 產品編號 | 是 | 對應 02_產品主檔 |
| 品名 | 是 | 產品名稱 |
| 計畫量 | 是 | 預計生產數量，必須大於 0 |
| 交期 | 是 | 預計完工日，YYYY-MM-DD |
| 工站名稱 | 否 | 建議工站 |
| 建議機台 | 否 | 建議機台編號 |
| 需求人力 | 否 | 預估人力 |
| 優先級 | 否 | 高 / 一般 / 低 |
| 來源 | 否 | 例如 5月計畫表 |
| 備註 | 否 | 補充說明 |

---

## 三、工單產生規則

若 CSV 未提供工單編號，系統自動產生：

```text
MO-YYYYMMDD-流水號
```

工單寫入：

```text
10_工單主檔
```

對應欄位：

| 工單欄位 | 來源 |
|---|---|
| 工單編號 | 自動產生或 CSV 工單編號 |
| 產品編號 | CSV 產品編號 |
| 品名 | CSV 品名 |
| 計畫量 | CSV 計畫量 |
| 已完成量 | 0 |
| 不良量 | 0 |
| 開始日 | 今天或 CSV 開始日 |
| 預計完工日 | CSV 交期 |
| 優先級 | CSV 優先級，預設一般 |
| 狀態 | 待生產 |
| 備註 | CSV 備註 |

---

## 四、排程需求池產生規則

排程需求池資料先產出 CSV，下一階段再改成直接寫入 Google Sheets。

對應欄位：

| 排程需求池欄位 | 來源 |
|---|---|
| 需求編號 | REQ-工單編號 |
| 來源 | CSV 來源，預設計畫清洗 |
| 產品編號 | CSV 產品編號 |
| 品名 | CSV 品名 |
| 需求量 | CSV 計畫量 |
| 交期 | CSV 交期 |
| 建議工站 | CSV 工站名稱 |
| 建議機台 | CSV 建議機台 |
| 需求人力 | CSV 需求人力 |
| 狀態 | 待排程 |
| 更新時間 | 系統產生 |

---

## 五、後續 v1.5.0 預計

- 檢查產品編號是否存在於 02_產品主檔。

# 24_派班報工每日結算

本階段目的：在 23_派班報工巡檢修復後，將每日派班與報工結果整理成日結資料，供 LINE 摘要、AI 戰情、主管看板使用。

## 正式檔案

```text
gas/24_派班報工每日結算正式模組.gs
gas/00_主程式_doPost_派班日結接入版_v24.gs
```

## API Action

```text
派班報工日結健康檢查
產生派班報工日結
取得派班報工日結
```

## 輸入資料來源

```text
10_今日派班表
09_派班報工紀錄
```

## 自動建立分頁

```text
24_派班報工日結
24_未報工追蹤
24_派班效率統計
24_派班報工日結紀錄
```

## 日結統計內容

```text
依 作業日 + 班別 + 工號 + 姓名 彙總：
- 派班筆數
- 已報工派班數
- 部分報工派班數
- 未報工派班數
- 派工量合計
- 良品數
- 不良數
- 報工數
- 剩餘量
- 完成率
- 不良率
- 狀態
```

## 未報工追蹤

未報工追蹤會列出今日派班表中尚未報工的資料，用於班長追蹤補報工或改派。

## 效率統計

依 作業日 + 班別 + 工站 + 工序 + 主機台 彙總，供後續 AI 戰情與產能分析使用。

## Apps Script 測試函數

```javascript
測試_派班報工日結健康檢查()
測試_產生派班報工日結()
測試_取得派班報工日結()
```

## 發佈步驟

```text
1. Apps Script 新增 / 更新 24_派班報工每日結算正式模組.gs
2. 主後端替換 doPost 接入版：00_主程式_doPost_派班日結接入版_v24.gs
3. 儲存
4. 部署 → 管理部署作業 → 編輯 Web App → 新增版本 → 部署
5. 先測 派班報工日結健康檢查
6. 執行 產生派班報工日結
7. 檢查 24_派班報工日結 / 24_未報工追蹤 / 24_派班效率統計
```

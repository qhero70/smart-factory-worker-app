# 21_今日派班報工對接正式模組

階段：10_今日派班表 → PWA 報工入口 → 報工扣工序 / 扣工單

## 目的

讓 PWA / LINE / Web API 可以直接使用 `10_今日派班表`：

```text
取得今日派班作業
寫入今日派班報工
回寫今日派班表
回寫工序明細
回寫工單主檔
建立派班報工紀錄
```

## 模組檔案

```text
gas/21_今日派班報工對接正式模組.gs
```

## 主要函數

```javascript
初始化_今日派班報工()
取得今日派班作業(payload)
寫入今日派班報工(payload)
今日派班報工_嘗試處理動作_(payload)

測試_初始化_今日派班報工()
測試_取得今日派班作業()
```

## 輸入來源

```text
10_今日派班表
```

## 輸出與回寫

```text
09_派班報工紀錄
09_派班報工操作紀錄
10_今日派班表
10_工單工序明細
10_工單主檔
```

## API 動作

接入既有主程式 `doPost(e)` 後，可用以下 action：

```text
派班報工健康檢查
初始化_今日派班報工
取得今日派班作業
寫入今日派班報工
```

## 主程式接入方式

在既有 `doPost(e)` 中，解析 payload 後、進入舊 API 分派前加入：

```javascript
var 派班報工結果 = 今日派班報工_嘗試處理動作_(p);
if (派班報工結果) return 輸出JSON_(派班報工結果);
```

若目前主程式使用 `排程需求池_輸出JSON_()`，也可改成：

```javascript
var 派班報工結果 = 今日派班報工_嘗試處理動作_(p);
if (派班報工結果) return 排程需求池_輸出JSON_(派班報工結果);
```

## 取得今日派班作業 payload

```json
{
  "action": "取得今日派班作業",
  "作業日": "2026-06-14",
  "工號": "hfn628",
  "班別": "早班"
}
```

`工號`、`姓名`、`班別`、`報工工站名稱` 都可選填。

## 寫入今日派班報工 payload

```json
{
  "action": "寫入今日派班報工",
  "今日派班編號": "TDSP-20260614-0001",
  "工號": "hfn628",
  "姓名": "黃寶全",
  "良品數": 30,
  "不良數": 0,
  "備註": "現場正常報工"
}
```

## 回寫規則

成功報工後：

```text
10_今日派班表.報工狀態 = 已報工 / 部分報工
10_今日派班表.現場確認 = 已確認
10_今日派班表.派班狀態 = 已完成 / 部分完成
10_工單工序明細.已完成量 += 良品數
10_工單工序明細.不良量 += 不良數
10_工單工序明細.剩餘量 = 計畫量 - 已完成量
10_工單主檔 依工序明細彙總回寫
09_派班報工紀錄 新增一筆
```

## 成功標準

1. 執行：

```javascript
測試_初始化_今日派班報工()
測試_取得今日派班作業()
```

2. `測試_取得今日派班作業()` 回傳今日可報工資料。
3. 用 API 或測試 payload 寫入報工後，`09_派班報工紀錄` 有資料。
4. `10_今日派班表` 狀態更新。
5. `10_工單工序明細` 數量更新。
6. `10_工單主檔` 彙總更新。

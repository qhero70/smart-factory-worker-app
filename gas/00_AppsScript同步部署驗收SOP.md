## 二、為什麼不用改 doPost(e)

正式主後端已經預留：

```javascript
if (typeof LINE身份權限_嘗試處理Webhook_ === 'function') {
  var 身份權限33結果 = LINE身份權限_嘗試處理Webhook_(p);
  if (身份權限33結果 && 身份權限33結果.已處理) {
    return 主程式_安全輸出JSON_(身份權限33結果);
  }
}
```

所以只要把 `LINE身份綁定.gs` 加進 Apps Script，主入口就會自動接住。

## 五、部署後 LINE 現場測試

### 1. 主檔檢查不可壞

輸入：

```text
主檔檢查
```

成功標準：仍能回覆主檔筆數。

### 2. 綁定測試

輸入：

```text
綁定 fhfi546
```

成功標準：LINE 回覆：

```text
綁定成功
工號：fhfi546
姓名：黃俊偉
角色：作業員
你現在可以使用：我的任務
```

同時 `33_LINE身份權限`：

| 欄位 | 期望 |
|---|---|
| A欄 LINE_USER_ID | 寫入真實 LINE userId |
| H欄 啟用 | 是 |
| I欄 備註 | LINE真實綁定完成 |

### 3. 我的任務測試

輸入：

```text
我的任務
```

成功標準：若該工號仍有待報工任務，回覆派班清單；若沒有，回覆目前沒有待報工任務。

### 4. 解除綁定測試

輸入：

```text
解除綁定
```

成功標準：`33_LINE身份權限` H欄改為 `否`，I欄改為 `已解除綁定；保留歷史資料`。

| 測試 | 預期 |

## 八、禁止事項

1. 禁止自編 LINE_USER_ID。
2. 禁止把工號當 LINE_USER_ID。
3. 禁止覆蓋 `doPost(e)`。
4. 禁止刪除既有 LINE 模組。
5. 禁止未部署新版本就宣稱 LINE 綁定已上線。

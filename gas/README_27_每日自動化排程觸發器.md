# 27_每日自動化排程觸發器

本階段目的：將 23 → 24 → 25 → 26 串成每日自動流程。

## 正式檔案

```text
gas/27_每日自動化排程觸發器正式模組.gs
gas/00_主程式_doPost_每日自動化接入版_v27.gs
```

## 每日排程

```text
20:10 自動執行：23_派班報工巡檢修復 → 24_派班報工每日結算 → 25_AI戰情摘要資料源
20:15 自動執行：26_LINE每日戰情推播
```

## API Action

```text
每日自動化健康檢查
初始化_每日自動化排程觸發器
建立_每日自動化排程觸發器
刪除_每日自動化排程觸發器
執行_每日戰情資料準備
執行_每日LINE戰情推播
執行_每日戰情全流程
```

## 自動建立分頁

```text
27_每日自動化排程紀錄
27_每日自動化觸發器設定
27_每日自動化錯誤紀錄
```

## Apps Script 測試函數

```javascript
測試_每日自動化健康檢查()
測試_初始化_每日自動化排程觸發器()
測試_建立_每日自動化排程觸發器()
測試_刪除_每日自動化排程觸發器()
測試_執行_每日戰情資料準備()
測試_執行_每日LINE戰情推播()
```

## 建立排程前必須確認

```text
23_派班報工巡檢修復正式模組.gs 已貼上
24_派班報工每日結算正式模組.gs 已貼上
25_AI戰情摘要資料源正式模組.gs 已貼上
26_LINE每日戰情推播正式模組.gs 已貼上
Script Properties 已設定 LINE_CHANNEL_ACCESS_TOKEN
Script Properties 已設定 LINE_TARGET_ID 或 LINE_TARGET_IDS
```

## 發佈步驟

```text
1. Apps Script 新增 27_每日自動化排程觸發器正式模組.gs
2. 主後端替換 doPost 接入版：00_主程式_doPost_每日自動化接入版_v27.gs
3. 儲存
4. 執行 測試_每日自動化健康檢查()
5. 執行 測試_初始化_每日自動化排程觸發器()
6. 執行 測試_建立_每日自動化排程觸發器()
7. 部署 → 管理部署作業 → 編輯 Web App → 新增版本 → 部署
```

## 驗收標準

```text
27_每日自動化觸發器設定 有兩列：資料準備、LINE推播
27_每日自動化排程紀錄 有建立觸發器紀錄
Apps Script 左側觸發條件 有兩個時間驅動觸發器
```

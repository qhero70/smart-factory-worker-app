## 九、錯誤與回滾方式

### 情境 A：LINE 無回覆

檢查：

1. `LINE_CHANNEL_ACCESS_TOKEN` 是否存在。
2. LINE Developers webhook 是否開啟。
3. Web App 存取權限是否為 `任何人`。
4. Apps Script 執行紀錄是否報錯。

### 情境 B：主檔檢查壞掉

回滾：

1. Apps Script 管理部署。
2. 回到上一個可用版本。
3. 不刪資料表。
4. 檢查是否誤改 `doPost(e)`。

### 情境 C：綁定錯人

處理：

1. 到 `33_LINE身份權限` 找該工號。
2. 將 H 欄 `啟用` 改為 `否`。
3. I 欄備註填 `人工停用：綁定錯誤`。
4. 正確人員重新輸入 `綁定 工號`。

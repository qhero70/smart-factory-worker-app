# 11_需求池轉工單正式模組

階段：10_排程需求池 → 10_工單主檔 / 訂單管理 / 拆工單

## 目標

將已完成清洗並寫入的 `10_排程需求池` 資料，轉成正式 `10_工單主檔`。

## 模組檔案

```text
gas/11_需求池轉工單正式模組.gs
```

## 主要函數

```javascript
初始化_10_工單主檔()
需求池_轉工單(payload)
測試_初始化_10_工單主檔()
測試_需求池轉工單()
```

## 輸入來源

```text
10_排程需求池
```

## 輸出目標

```text
10_工單主檔
10_工單轉換紀錄
```

## 防重複規則

以 `來源需求編號` 對應 `10_排程需求池.需求編號`。

若 `10_工單主檔` 已存在同一個來源需求編號，則略過，不重複建立工單。

## 回寫規則

成功建立工單後，回寫 `10_排程需求池`：

```text
狀態 = 已轉工單
備註 = 已產生工單：WO-xxxxxxxx
更新時間 = 轉換時間
```

## 工單初始狀態

新工單狀態：

```text
待排程
```

後續流程：

```text
待排程 → 已排程 → 已派工 → 生產中 → 已完成 / 異常暫停
```

## 使用方式

先在 Apps Script 執行：

```javascript
測試_初始化_10_工單主檔()
```

再執行：

```javascript
測試_需求池轉工單()
```

成功後檢查：

```text
10_工單主檔
10_工單轉換紀錄
10_排程需求池 狀態欄
```

## 階段標準

成功標準：

- `10_工單主檔` 有新增工單。
- `10_工單轉換紀錄` 有批次紀錄。
- `10_排程需求池` 已轉工單的資料狀態回寫為 `已轉工單`。
- 重複執行不會重複產生工單。

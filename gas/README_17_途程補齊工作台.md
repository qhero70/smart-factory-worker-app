# 17_途程補齊工作台正式模組

階段：10_途程缺口清單 → 08_工站途程補齊工作台 → 08_工站途程機台主檔

## 目標

將目前 `10_途程缺口清單` 轉成工程可填寫、可確認、可寫回的標準工作台。

## 模組檔案

```text
gas/17_途程補齊工作台正式模組.gs
```

## 主要函數

```javascript
初始化_08_工站途程補齊工作台()
途程補齊_產生工作台(payload)
途程補齊_寫回途程主檔(payload)
途程補齊_重拆前清理(payload)

測試_初始化_08_工站途程補齊工作台()
測試_途程補齊產生工作台()
測試_途程補齊寫回途程主檔()
測試_途程補齊重拆前清理()
```

## 輸入來源

```text
10_途程缺口清單
08_工站途程機台主檔
```

## 輸出目標

```text
08_工站途程補齊工作台
10_途程補齊操作紀錄
```

## 工作台用途

工程端在 `08_工站途程補齊工作台` 補齊或確認以下欄位：

```text
途程順序
報工工站名稱
工序清單
機台編號清單
主機台
區域清單
需求人力
標準產能
標準工時_秒
填寫狀態
```

## 填寫狀態規則

可用值：

```text
待填寫
待確認
已確認
待寫回
不寫回
```

只有以下狀態會被寫回 `08_工站途程機台主檔`：

```text
已確認
待寫回
```

## 寫回規則

寫回主檔時會檢查必要欄位：

```text
產品編號
報工工站名稱
工序清單
機台編號清單
```

不足資料會略過，不會寫入錯誤途程。

## 防重複規則

以以下欄位組合防重複：

```text
產品編號 + 客戶品號 + 品名 + 途程順序 + 報工工站名稱 + 工序清單 + 機台編號清單
```

## 重拆流程

途程寫回後，執行：

```javascript
測試_途程補齊重拆前清理()
```

這會：

```text
清空 10_工單工序明細
清空 10_拆工單紀錄
將已拆工單狀態退回 待拆工單
```

然後再執行：

```javascript
測試_拆工單()
```

## 階段封關標準

成功標準：

- `08_工站途程補齊工作台` 產生缺口資料。
- 工程端把可用資料改為 `已確認` 或 `待寫回`。
- `測試_途程補齊寫回途程主檔()` 成功寫入 `08_工站途程機台主檔`。
- `測試_途程補齊重拆前清理()` 後重新 `測試_拆工單()`。
- `10_拆工單紀錄` 的 `無途程工單` 明顯下降，目標為 0。

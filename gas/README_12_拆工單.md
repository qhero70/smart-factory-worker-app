# 12_拆工單正式模組

階段：10_工單主檔 → 拆工單 → 工站工序派工明細

## 目標

將 `10_工單主檔` 的正式工單，依 `08_工站途程機台主檔` 拆成可排程、可派工、可報工扣工單的工序明細。

## 模組檔案

```text
gas/12_拆工單正式模組.gs
```

## 主要函數

```javascript
初始化_10_工單工序明細()
工單_拆工序(payload)
測試_初始化_10_工單工序明細()
測試_拆工單()
```

## 輸入來源

```text
10_工單主檔
08_工站途程機台主檔
```

## 輸出目標

```text
10_工單工序明細
10_拆工單紀錄
```

## 防重複規則

以 `工單編號` 判斷是否已存在於 `10_工單工序明細`。

同一張工單已拆過時，重複執行會略過，不會重複產生工序明細。

## 途程對應規則

依序嘗試匹配：

1. 產品編號 + 客戶品號 + 品名
2. 產品編號
3. 客戶品號
4. 品名

若找不到途程，建立一筆預設工序：`未設定途程 / 待工程補途程`，並列入 `無途程工單` 數。

## 回寫規則

成功拆工單後，回寫 `10_工單主檔`：

```text
狀態 = 已拆工單
備註 = 已拆工序：N 道
更新時間 = 拆分時間
```

## 工序狀態規則

第一道工序：

```text
待排程
```

第二道以後：

```text
等待前工序
```

## 使用方式

先在 Apps Script 執行：

```javascript
測試_初始化_10_工單工序明細()
```

再執行：

```javascript
測試_拆工單()
```

成功後檢查：

```text
10_工單工序明細
10_拆工單紀錄
10_工單主檔 狀態欄
```

## 階段標準

成功標準：

- `10_工單工序明細` 有新增工序資料。
- `10_拆工單紀錄` 有批次紀錄。
- `10_工單主檔` 已拆工單資料狀態為 `已拆工單`。
- 重複執行不會重複產生工序明細。

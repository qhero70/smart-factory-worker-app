# 19_派班需求與自動派班正式模組

階段：10_工單工序明細 → 10_派班需求池 → 10_每日派班表

## 目的

途程補齊先暫停，主線改以「派班可用」為目的。

本模組先把已拆出的有效工序轉成可派班資料，再依人員主檔與工站關聯資料自動派班。

## 模組檔案

```text
gas/19_派班需求與自動派班正式模組.gs
```

## 主要函數

```javascript
初始化_10_派班系統()
派班_產生需求池(payload)
派班_自動派班(payload)

測試_初始化_10_派班系統()
測試_產生派班需求池()
測試_自動派班()
```

## 輸入來源

```text
10_工單工序明細
01_人員主檔
04_工站關聯資料 / 04_工站關聯 / 04_工站關聯寫入核檢 / 員工工站排班表
```

## 輸出目標

```text
10_派班需求池
10_每日派班表
10_派班紀錄
10_派班參數設定
```

## 派班需求池產生規則

只納入：

```text
有效工序
剩餘量 > 0
不是 已完成 / 取消 / 已派班
報工工站名稱不是 未設定途程
工序不是 待工程補途程
```

排除途程缺口資料，避免錯派。

## 派班排序

依：

```text
派班日期
交期風險
優先級
```

從早到晚、風險高者優先。

## 自動派班規則

1. 先依產品編號 + 客戶品號 + 工站找候選人員。
2. 找不到時降階為產品編號 + 工站、客戶品號 + 工站、工站。
3. 仍找不到時使用同班別可用人員。
4. 同一人同一日期同一班別預設只派 1 筆。
5. 同一機台同一日期同一班別預設只派 1 筆。
6. 找不到人或機台已滿時，標記為 `待人工派班`。

## 成功標準

執行：

```javascript
測試_初始化_10_派班系統()
測試_產生派班需求池()
測試_自動派班()
```

檢查：

```text
10_派班需求池：有待派班 / 已派班 / 待人工派班資料
10_每日派班表：有日期、班別、工號、姓名、工站、工單、機台
10_派班紀錄：有批次紀錄
```

# 10_排程需求池 GAS 接入說明

## 目的

讓 `docs/production-plan-cleaner-v3.html` 的「寫入 10_排程需求池」按鈕，可以把資料正式寫入 Google Sheets。

## 已新增正式後端模組

檔案：

```text
gas/10_排程需求池後端.gs
```

此模組會自動建立並維護：

```text
10_排程需求池
10_生產計畫清洗紀錄
10_排程參數設定
```

## 接入方式 A：目前 GAS 專案沒有 doPost

可以直接貼上整份 `10_排程需求池後端.gs`。

可用動作：

```text
健康檢查
初始化_10_排程需求池
寫入排程需求池
```

## 接入方式 B：目前 GAS 專案已經有 doPost

不要新增第二個 `doPost`。

請保留你原本的 LINE Bot / 報工 doPost，然後在原本 doPost 解析 payload 後，插入一段分派判斷：

```text
呼叫：排程需求池_嘗試處理動作_(payload)
若有回傳結果，就直接輸出 JSON 並結束。
```

這樣可以避免破壞既有 LINE Bot、主檔檢查與報工 API。

## 初始化測試

在 Apps Script 編輯器中先執行：

```text
初始化_10_排程需求池()
```

成功後，Google Sheets 會出現：

```text
10_排程需求池
10_生產計畫清洗紀錄
10_排程參數設定
```

## 前端測試

開啟：

```text
https://qhero70.github.io/smart-factory-worker-app/production-plan-cleaner-v3.html?v=300
```

上傳生產計畫表後，按：

```text
寫入 10_排程需求池
```

成功時前端會顯示：

```text
寫入 X 筆，略過重複/無效 Y 筆
```

## 防重複規則

優先使用前端傳來的 `唯一鍵`。

若前端沒傳唯一鍵，後端會自動用以下欄位建立：

```text
來源工作表 + 產品編號 + 客戶品號 + 品名 + 需求日期 + 類型 + 數量
```

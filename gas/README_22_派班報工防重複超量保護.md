# 22_派班報工防重複超量保護正式模組

本模組是 PWA 今日派班報工的安全保護層。

## 正式檔案

```text
gas/22_派班報工防重複超量保護.gs
gas/00_主程式_doPost_派班報工防呆接入版.gs
```

## 接入原則

Apps Script 專案內只能保留一個 `doPost(e)`。  
請用 `00_主程式_doPost_派班報工防呆接入版.gs` 的內容取代主後端原本的 `doPost(e)` 與 `處理API請求_(action, p)`。

## doPost 分流順序

```text
1. 派班報工防呆
2. 今日派班報工
3. 排程需求池
4. LINE Webhook
5. 既有 API
```

## 防呆規則

```text
1. 今日派班報工狀態 = 已報工，不允許再次扣量。
2. 派班狀態 = 已完成，不允許再次扣量。
3. 120 秒內同一派班、同一人員、同一良品數、同一不良數，視為連點或重送。
4. 良品數不可超過 10_工單工序明細.剩餘量。
```

## 測試函數

```javascript
測試_派班報工防呆健康檢查()
```

正常回傳：

```json
{
  "ok": true,
  "模組": "22_派班報工防重複超量保護",
  "版本": "v22.1.0"
}
```

## 發佈步驟

```text
1. Apps Script 新增 / 更新 22_派班報工防重複超量保護.gs
2. 主後端替換 doPost(e)、處理API請求_(action, p)、主程式_安全輸出JSON_(obj)
3. 儲存
4. 部署 → 管理部署作業 → 編輯 Web App → 新增版本 → 部署
5. 回 PWA 測試派班報工
```

## 驗收標準

```text
09_派班報工操作紀錄 出現：寫入今日派班報工_防呆版
超量報工會被阻擋：OVER_REMAINING_QTY
連點重送會被阻擋：RECENT_DUPLICATE_REPORT
已完成派班重送會被阻擋：DUPLICATE_DISPATCH_REPORT
```

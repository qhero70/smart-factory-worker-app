# V5 報工扣工單測試標準

## 一、先確認 GAS 已有檔案

中央資料庫 GAS 專案內要有：

```text
中央資料庫API_v3_APS獨立版.gs
中央資料庫API_v4_派班附加模組.gs
中央資料庫API_v5_報工扣工單附加模組.gs
```

## 二、doPost 必須支援

```text
執行報工扣工單V1
寫入現場報工V1
```

## 三、第一次測試

如果 `19_現場報工回寫` 還沒有資料，執行：

```javascript
執行報工扣工單V1_({})
```

成功標準：

```text
20_工單進度追蹤：有資料
21_完工入庫建議：可能為 0 筆
22_報工異常警示：可能為 0 筆
報工筆數：0
```

## 四、寫入一筆測試報工

在 Apps Script 執行區改測：

```javascript
寫入現場報工V1_({
  工單號:'請填 13_拆工單結果 裡面任一工單號',
  工號:'TEST001',
  姓名:'測試人員',
  工站:'加工',
  機台:'測試機台',
  良品數:10,
  不良數:1,
  工時_分鐘:60,
  來源:'手動測試'
})
```

再執行：

```javascript
執行報工扣工單V1_({})
```

成功標準：

```text
19_現場報工回寫：新增 1 筆
20_工單進度追蹤：該工單累計良品 +10、不良 +1
22_報工異常警示：如果有不良，會出現發生不良警示
```

## 五、正式用途

後續會讓 PWA 報工表單直接寫入：

```text
19_現場報工回寫
```

再由扣工單引擎更新：

```text
20_工單進度追蹤
21_完工入庫建議
22_報工異常警示
```

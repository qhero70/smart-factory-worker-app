## 4. 報工作業 V4 寫入規則鎖定

### 4.1 正常報工

固定 append 到：

```text

```

### 4.2 有不良時

```text

```


## 5. 不良原因來源規則

不良原因固定來源：

```text
05_不良代碼主檔
```

前端必須顯示：

```text
不良代碼
中文不良名稱
英文不良名稱
```

固定規則：

```text
不良分配數量總和不可超過 Step 3 的不良數
若 Step 3 不良數大於 0，Step 4 至少要有一筆不良原因分配
有輸入不良分配數量時，必須選擇不良原因
不良原因資料必須由 05_不良代碼主檔 對接
不良原因必須包含中文 + 英文
不得使用前端假資料
```

## 6. 工站、工序、機台來源規則

工站與機台固定來源：

```text
08_工站途程機台主檔
```

`08_工站途程機台主檔` 是產品途程資料，不是工站能力總表。

機台選擇不可使用「工站能力總機台清單」亂補。

固定階層：

```text
產品 / 品名
→ 工站
→ 工序 OP
→ 機台清單
```

選完工站後，主機台下拉與機台卡片必須使用 `08_工站途程機台主檔` 的「機台清單」欄位，不是只顯示「主機台」。

## 8. UI / UX / 報工邏輯固定

後續修改不得破壞：

```text
1. 滑動人員卡、產品卡不誤觸選取。
2. 真正短按才選取。
3. 人員掃碼圖示在搜尋框右側。
4. 產品掃碼圖示在搜尋框右側。
5. 掃碼功能不得另外佔一大排。
6. 未選人員前，不顯示選定人員卡。
7. 選完人員後，選定人員卡彈到中間。
8. 未選產品前，不顯示產品資訊、工站、主機台。
9. 選完產品後，才顯示產品資訊與工站。
10. 選完工站後，才顯示工序、標準產能、標準工時、主機台、機台卡片。
11. 主機台與機台卡片必須吃 08_工站途程機台主檔 的機台清單。
12. 人員、產品、機台都要顯示照片。
13. 有照片顯示照片，沒照片用小型佔位。
14. 人員、產品、機台卡片照片要滿版。
15. 人員、產品、機台卡片文字要在照片上，且字體要粗體。
16. 沒照片時，不得用大字「無機台照」撐滿卡片。
17. 黃色提示框縮小。
18. Step 3 / Step 4 / Step 5 不得超出手機表單外。
19. 產出數量欄位、時間欄位、異常時間框、照片按鈕、底部按鈕、搜尋列、提示框都必須符合手機寬度。
20. 不良分配數量不可超過 Step 3 的不良數。
21. 送出後必須 append 到 09_報工，不得跳到空白列後面。
22. 有不良時必須同步 append 到 09_不良紀錄。
23. 不再預填 0。
24. 頂覽說明要凍結。
25. 掃描器要能自動偵測功能；不支援時保留掃描槍與手動輸入。
```

## 9. 顯示流程鎖定

### 9.1 還沒選人員

```text
不顯示選定人員卡
```

### 9.2 選完人員

```text
顯示選定人員卡
選定人員卡彈出並置中
```

### 9.3 還沒選產品

```text
不顯示 selectedProductArea
不顯示產品資訊
不顯示工站
不顯示主機台
```

### 9.4 選完產品

```text
顯示 selectedProductArea
顯示 workstationSelect
```

### 9.5 還沒選工站

```text
隱藏 processRange
隱藏 stdCapacity
隱藏 stdTimeSec
隱藏 mainMachineSelect
隱藏 machineListGrid
```

### 9.6 選完工站

```text
顯示 processRange
顯示 stdCapacity
顯示 stdTimeSec
顯示 mainMachineSelect
顯示 machineListGrid
顯示機台照片
```

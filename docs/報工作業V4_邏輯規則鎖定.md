## 1. 正式架構鎖定

後續 PWA 只對接此正式 GAS Web App，不再切換其他 URL。

正式 GAS 專案只保留以下 3 支 `.gs`：

1. `正式主資料庫API_入口_只保留doGet_doPost.gs`
2. `智慧製造_PWA_API.gs`
3. `報工作業V4_正式寫入_09報工_09不良紀錄.gs`

其餘舊版報工作業、測試版、重複版、V1 / V2 / V3 / V5 報工模組，全部視為舊資料，不再使用。

## 2. 資料庫寫入規則

### 2.1 正常報工

正常報工固定 append 到：

- `09_報工`

### 2.2 有不良時

有不良時必須同步 append 到：

- `09_報工`
- `09_不良紀錄`

### 2.3 不良原因來源

不良原因固定來源：

- `05_不良代碼主檔`

前端選單必須顯示：

- 不良代碼
- 中文不良名稱
- 英文不良名稱

不得只顯示代碼，也不得只顯示中文。

---

## 3. 工站、工序、機台來源規則

工站與機台固定來源：

- `08_工站途程機台主檔`

機台選擇不可使用「工站能力總機台清單」亂補。

固定階層：

```text
產品 / 品名
→ 工站
→ 工序 OP
→ 機台清單
```

主機台下拉與機台卡片必須使用 `08_工站途程機台主檔` 的「機台清單」欄位。

## 5. UI / UX 固定規則

1. 滑動人員卡片、產品卡片時，不得誤觸選取。
2. 真正短按人員卡片、產品卡片時，才可選取。
3. 人員掃碼小相機圖示必須放在人員搜尋框右側。
4. 產品掃碼小相機圖示必須放在產品搜尋框右側。
5. 掃碼功能不得另外佔一大排。
6. 還沒選人員前，不顯示選定人員卡。
7. 選完人員後，選定人員卡才彈出並置中。
8. 還沒選產品前，不顯示產品資訊、工站、主機台區域。
9. 選完產品後，才顯示產品資訊與工站選擇。
10. 還沒選工站前，隱藏工序 OP、標準產能、標準工時秒、主機台、機台卡片。
11. 選完工站後，才顯示工序 OP、標準產能、標準工時秒、主機台、機台卡片。
12. 選完工站後，主機台與機台卡片必須顯示機台照片。
13. 人員、產品、機台卡片照片要滿版。
14. 人員、產品、機台卡片文字要壓在照片上，且字體要粗體。
15. 沒照片時，只能顯示小型佔位，不得用大字撐滿卡片。
16. 黃色提示框要縮小，避免壓縮手機畫面。
17. Step 3、Step 4、Step 5 不得超出手機表單外。
18. 產出數量欄位、時間欄位、異常時間框、照片按鈕、底部按鈕、搜尋列、提示框，都必須符合手機寬度。
19. 不再預填 0。
20. 頂覽說明要凍結，滑動時仍保留狀態資訊。
21. 掃描器要能自動偵測功能；不支援相機條碼偵測時，必須保留手動輸入與掃描槍輸入。

---

## 6. 不良分配固定規則

1. 不良分配數量總和不可超過 Step 3 的不良數。
2. 若 Step 3 不良數大於 0，Step 4 至少要有一筆不良原因分配。
3. 有輸入不良分配數量時，必須選擇不良原因。
4. 不良原因資料必須由 `05_不良代碼主檔` 對接。
5. 不良原因必須包含中文 + 英文。

---

## 7. 後續修改禁令

後續修改不得破壞以下行為：

- 不得改回大按鈕掃碼佔一整排。
- 不得改回滑動卡片會誤觸選取。
- 不得在未選產品前顯示產品資訊 / 工站 / 主機台。
- 不得在未選工站前顯示工序 / 主機台 / 機台卡片。
- 不得用工站能力總機台清單替代產品途程的機台清單。
- 不得讓不良分配數量超過 Step 3 不良數。
- 不得讓報工資料跳到空白列後面。
- 不得移除 `09_不良紀錄` 同步寫入規則。
- 不得再切換其他 GAS Web App URL。

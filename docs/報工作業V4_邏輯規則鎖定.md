## 2. 資料庫寫入規則

### 2.1 正常報工

正常報工固定 append 到：

- `09_報工`

### 2.2 有不良時

- `09_報工`

## 3. 不良原因來源規則

不良原因固定來源：

- `05_不良代碼主檔`

前端選單必須顯示：

- 不良代碼
- 中文不良名稱
- 英文不良名稱

不得只顯示代碼，也不得只顯示中文。

不良分配固定規則：

1. 不良分配數量總和不可超過 Step 3 的不良數。
2. 若 Step 3 不良數大於 0，Step 4 至少要有一筆不良原因分配。
3. 有輸入不良分配數量時，必須選擇不良原因。
4. 不良原因資料必須由 `05_不良代碼主檔` 對接。
5. 不良原因必須包含中文 + 英文。

## 4. 工站、工序、機台來源規則

工站與機台固定來源：

- `08_工站途程機台主檔`

`08_工站途程機台主檔` 是產品途程資料，不是工站能力總表。

機台選擇不可使用「工站能力總機台清單」亂補。

固定階層：

```text
產品 / 品名
→ 工站
→ 工序 OP
→ 機台清單
```

選完工站後，主機台下拉與機台卡片必須使用 `08_工站途程機台主檔` 的「機台清單」欄位，不是只顯示「主機台」。

## 6. UI / UX 固定規則

1. 滑動人員卡片、產品卡片時，不得誤觸選取。
2. 真正短按人員卡片、產品卡片時，才可選取。
3. 人員掃碼小相機圖示必須放在人員搜尋框右側。
4. 產品掃碼小相機圖示必須放在產品搜尋框右側。
5. 掃碼功能不得另外佔一大排。
6. 還沒選人員前，不顯示選定人員卡。
7. 選完人員後，選定人員卡才彈出並置中。
8. 還沒選產品前，不顯示產品資訊、工站、主機台區域。
9. 選完產品後，才顯示產品資訊與工站選擇。
10. 還沒選工站前，隱藏：
    - `processRange`
    - `stdCapacity`
    - `stdTimeSec`
    - `mainMachineSelect`
    - `machineListGrid`
11. 選完工站後，才顯示：
    - 工序 OP
    - 標準產能
    - 標準工時秒
    - 主機台
    - 機台卡片
12. 選完工站後，主機台與機台卡片必須顯示機台照片。
13. 人員、產品、機台卡片照片要滿版。
14. 人員、產品、機台卡片文字要壓在照片上，且字體要粗體。
15. 有照片時必須顯示照片。
16. 沒照片時，只能顯示小型佔位，不得用大字「無機台照」或「無產品照」撐滿卡片。
17. 黃色提示框要縮小，避免壓縮手機畫面。
18. Step 3、Step 4、Step 5 不得超出手機表單外。
19. 產出數量欄位、時間欄位、異常時間框、照片按鈕、底部按鈕、搜尋列、提示框，都必須符合手機寬度。
20. 不再預填 0。
21. 頂覽說明要凍結，滑動時仍保留狀態資訊。
22. 掃描器要能自動偵測功能；不支援相機條碼偵測時，必須保留手動輸入與掃描槍輸入。

## 7. 手機觸控防誤觸規則

滑動產品卡 / 人員卡：

```text
滑動 → 不觸發選取
真正短按 → 才觸發選取
```

不得讓產品卡、人员卡的 `touchstart / pointerdown` 直接觸發選取。

選取判斷必須以位移距離與按壓時間判定：

1. 若手指移動距離超過安全閾值，視為滑動。
2. 若是滑動，不得選取。
3. 若是短按，才允許選取。


```text
不顯示 selectedProductArea

```

### 8.4 選完產品

```text
顯示 selectedProductArea
顯示 workstationSelect
```

### 8.5 還沒選工站

```text
隱藏 processRange
隱藏 stdCapacity
隱藏 stdTimeSec
隱藏 mainMachineSelect
隱藏 machineListGrid
```

### 8.6 選完工站

```text
顯示 processRange
顯示 stdCapacity
顯示 stdTimeSec
顯示 mainMachineSelect
顯示 machineListGrid
顯示機台照片
```

## 9. 後續修改禁令

後續修改不得破壞以下行為：

- 不得改回大按鈕掃碼佔一整排。
- 不得改回滑動卡片會誤觸選取。
- 不得在未選人員前顯示選定人員卡。
- 不得在未選產品前顯示產品資訊 / 工站 / 主機台。
- 不得在未選工站前顯示工序 / 主機台 / 機台卡片。
- 不得用工站能力總機台清單替代產品途程的機台清單。
- 不得讓不良分配數量超過 Step 3 不良數。
- 不得讓報工資料跳到空白列後面。
- 不得移除 `09_不良紀錄` 同步寫入規則。
- 不得再切換其他 GAS Web App URL。
- 不得把報工 V4 改成 V1 / V2 / V3 / V5 的資料流。
